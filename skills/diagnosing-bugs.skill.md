# Diagnosing Bugs — 系统性 Bug 诊断

> 完整内容已安装为 Cowork skill。
> 当用户说"调试""诊断""修 bug""排查"，或报告异常/报错/性能退化时触发。

## 6 阶段调试循环

```
阶段 1: 反馈环     → 构建最短可复现路径
阶段 2: 复现       → 稳定触发 bug
阶段 3: 假设排序   → 列出所有可能根因，按概率排序
阶段 4: 探针       → 最简产出来验证/排除每个假设
阶段 5: 修复+回归  → TDD 方式修复（先写失败测试）
阶段 6: 清理       → 文档化根因 + 更新记忆
```

## 在 auto-exec 中的角色

- **止血型 bug**（根因明确、≤3 文件）→ 诊断后当前会话修复
- **排查型 bug**（根因不确定）→ diagnostics skill 完成阶段 1-3 输出假设，然后走 auto-exec 调度
