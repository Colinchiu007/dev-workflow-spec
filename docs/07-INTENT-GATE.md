# Intent Gate — 意图分类门禁

## 是什么

接到请求后，先快速分类再决定行动路径。防止对模糊需求过度执行，也防止对复杂任务草率动手。

## 分类表

| 类型 | 特征 | 行动 |
|------|------|------|
| **简单查询** | 单一文件/已知路径/直接问题 | 直接回答，不绕弯 |
| **明确指令** | 具体文件+具体操作 | 评估复杂度：>30min 或 ≥3 步骤 → 走 auto-exec；否则直接执行 |
| **调研探索** | "怎么实现的""在哪里" | 先搜代码再答，可并行搜多个角度 |
| **开放任务** | "优化一下""重构""加个功能" | 先读代码评估现状：>30min 或 ≥3 步骤 → 走 auto-exec；否则出方案动手 |
| **模糊需求** | 意图不清、多种理解、范围不明 | 先反问确认，不确定就说不确定 |

## 工作流

```
收到请求
  │
  ├── 简单查询 → 直接回答
  │
  ├── 明确指令 → 评估复杂度
  │   ├── >30min / ≥3 步骤 → auto-exec
  │   └── 否则 → 直接执行
  │
  ├── 调研探索 → 搜索代码 → 回答
  │
  ├── 开放任务 → 读代码评估
  │   ├── >30min / ≥3 步骤 → auto-exec
  │   └── 否则 → 出方案 → 确认 → 动手
  │
  └── 模糊需求 → 反问澄清 → 再分类
```

## auto-exec 强制触发条件

分类后，如果预估工作量 > 30 分钟或需要 3+ 个步骤且步骤间有依赖关系，**禁止在当前会话直接执行全部工作**。必须走 auto-exec 编排。

### 豁免条件

| 场景 | 处理 |
|------|------|
| 用户明确要求不走编排 | 可以豁免，但注明上下文溢出风险 |
| 止血型 bug（根因明确、≤3 文件、5-10 分钟） | 直接修复 |
| 简单查询、一次性能搞完 | 不触发 |
