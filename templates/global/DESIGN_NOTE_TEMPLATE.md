# Design Note 模板

> 提交至 `.cowork/design-notes/DN-NNN-short-description.md`

## 1. Why — 为什么要做

用一段话说明业务/技术动机。

## 2. What — 做什么

### 2.1 变更范围

| 变更项 | 类型 | 说明 |
|--------|------|------|
| `path/to/file` | 新增/修改/删除 | 简短说明 |

### 2.2 共享契约变更

- [ ] 新增模型
- [ ] 修改现有模型
- [ ] 无变更

## 3. How — 怎么做

### 3.1 方案概述

3-5 句话描述思路。

### 3.2 失败处理

什么情况下会失败，降级方案。

### 3.3 回滚方式

回滚步骤。

## 4. Verification — 怎么验证

| ID | 标准 | 通过条件 |
|----|------|---------|
| AC-01 | 描述 | exit code 0 |

### 自验

```
□ 验收标准全部 PASS
□ .clinerules 未被违反
□ 全量测试 → PASS
```
